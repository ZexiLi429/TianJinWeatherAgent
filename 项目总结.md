# 天津交通气象哨兵 · 项目总结

> 项目名称：天津交通实时气象综合服务平台（FengHe-Weather）  
> 开发周期：2025 年 ~ 2026 年  
> 生产地址：https://tianjin-weather-sentinel.pages.dev

---

## 一、项目概述

本项目是一个面向天津市市民及交通出行者的气象综合服务 Web 应用。平台整合实时气象数据、气象预警、积涝监测、交通路况、AI 智能问答等功能，提供一站式气象出行服务。核心目标是让用户在出行前快速了解天气状况、规避积涝和极端天气风险。

## 三、技术架构

### 前端技术栈

```
React 19 + TypeScript 5.8
Vite 6（构建工具）
Tailwind CSS 4（样式）
Framer Motion（动画）
React-Leaflet + Leaflet 1.9（地图）
Recharts（图表）
Lucide React（图标）
```

### 数据来源

| 数据类型 | 来源 |
|---------|------|
| 实时天气 / 逐小时预报 | 和风天气 API（QWeather） |
| 气象预警 | 和风天气 API + 国家气象中心（NMC） |
| 降水实况 | 和风天气分钟级降水 |
| 地图底图 & 路况 | 高德地图 API |
| 地铁线路 | 高德地图地铁 API |
| AI 问答 | 智谱 GLM-4 / Google Gemini（可切换） |

### 部署架构

```
本地开发  →  npm run dev（Vite 开发服务器，端口 3000）
             ↓ API 代理（/api/qweather、/api/amap、/api/nmc）
生产构建  →  npm run build → dist/
             ↓
部署      →  Cloudflare Pages（Wrangler 4）
             https://tianjin-weather-sentinel.pages.dev
```

## 四、核心功能设计亮点

### 4.1 AI 智能助手（津小天 / 津小晴）

- 随机出现男娃（津小天）或女娃（津小晴）角色，常驻悬浮，不消失
- 气泡台词每 6 秒轮播，男女角色各有 10 条专属语境台词
- 可拖拽定位，点击打开全屏 AI 对话界面
- 对话界面中 AI 回复头像为对应角色图片

### 4.2 暴雨积涝预警系统

- 涵盖天津 16 处历史易涝点，含立交桥底、下穿隧道、低洼路段
- 实时水位监测 + 风险等级（低/中/高）三色预警
- 接入和风天气 API 获取真实降水数据和气象预警
- 积水地图通过 Leaflet 动态渲染，颜色按风险等级区分
- 地图 Tab 切换 Bug 修复：切换 Tab 后销毁 Leaflet 实例，返回后重建

### 4.3 实时交通路况

- 高德地图路况叠加显示，颜色区分畅通/缓行/拥堵
- 天津地铁 5 条主线运营状态，含实时班次信息
- 天气-路况联动分析，暴雨天自动触发风险提示

### 4.4 气象预警地图

- 分层叠加气象预警区域（红/橙/黄/蓝四级）
- 展示预警发布时间、影响区域、建议措施
- 自动计算预警距发布经过时长

## 五、关键技术问题与解决方案

### 5.1 Leaflet 地图 Tab 切换后不显示

**问题**：在积涝预警页面，切换到"数据统计"或"积水列表"Tab 后返回"地图总览"，地图空白。

**原因**：Leaflet 地图实例挂载在 `mapRef.current` 中，切换 Tab 时容器 DOM 隐藏但实例未销毁，`useEffect` 中的 `if (mapRef.current) return` 导致无法重新初始化。

**解决方案**：监听 `activeTab` 变化，离开地图 Tab 时主动调用 `mapRef.current.remove()` 销毁实例并清空 ref；返回时重新创建。

```typescript
useEffect(() => {
  if (activeTab !== 'map') {
    mapRef.current?.remove();
    mapRef.current = null;
    return;
  }
  // 重新初始化地图...
}, [activeTab, floodPoints]);
```

### 5.2 和风天气 API 403 Invalid Host

**问题**：直接通过 Vite 代理转发请求时，API 返回 `403 Invalid Host`。

**原因**：和风天气 API 校验请求的 `Referer` 和 `Origin` 头，浏览器默认不发送或发送错误来源。

**解决方案**：在 `vite.config.ts` 的代理配置中手动注入合法的 `Referer` 和 `Origin` 头。

```typescript
'/api/qweather': {
  target: 'https://api.qweather.com',
  changeOrigin: true,
  headers: {
    'Referer': 'https://dev.qweather.com/',
    'Origin': 'https://dev.qweather.com',
  },
}
```

### 5.3 Windows PowerShell npm 脚本执行受限

**问题**：Windows 环境下运行 `npm run build` 报错 `ps1 脚本无法执行`。

**解决方案**：每次开启终端前执行：
```powershell
Set-ExecutionPolicy Bypass -Scope Process -Force
```
